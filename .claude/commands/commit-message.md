## Context:
- Current git status: !`git status`
- Current git diff: !`git diff --staged`

В первую очередь выполни команду `git add *` для внесения всех изменений в staged.
Проанализируй подготовленные (staged) изменения в git и создай сообщение коммита. Используй настоящее время и объясняй, "почему" были внесены изменения, а не только "что" изменилось.

## Commit types

Используются следующие типы коммитов:

- `feat:` — новая функциональность  
- `fix:` — исправление ошибок  
- `refactor:` — рефакторинг кода без изменения поведения  
- `docs:` — изменения в документации  
- `style:` — изменения форматирования (не влияющие на логику)  
- `test:` — добавление или изменение тестов  
- `perf:` — улучшение производительности  
- `chore:` — технические изменения (сборка, зависимости, конфиги, инфраструктура)

## Format:
Используй следующий формат для сообщения коммита:
```
<type>: <краткое_описание>
<необязательное_описание_почему>
```

## Output:

1. Покажи краткое описание текущих staged-изменений  
2. Предложи сообщение коммита с подходящим эмоджи  
3. Запроси подтверждение перед выполнением коммита  

НЕ выполняй коммит автоматически — дождись подтверждения пользователя и выполняй коммит только после этого.
