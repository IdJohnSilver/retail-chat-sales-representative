# План реализации: Chat List

Основан на спецификации: [_specs/chat-list.md](../_specs/chat-list.md)

---

## Шаг 1. Обновить тип Chat

**Цель:** привести тип `Chat` в соответствие с форматом ответа API.

- Формат API: `{ chaCode, chaId, chaName }`
- Текущий тип `Chat` содержит поля `id`, `tradePointId`, `participants`, `lastMessage`, которые не соответствуют реальному API
- Обновить интерфейс `Chat` в `src/types/index.ts`, чтобы он отражал структуру ответа сервера

**Файлы:**
- `src/types/index.ts` (изменение)

---

## Шаг 2. API-функция получения чатов

**Цель:** реализовать вызов `GET chats` через httpClient.

- Создать `src/api/chatsApi.ts` с функцией `fetchChats()`
- Функция отправляет `GET` на `chats` через `httpClient` (заголовок авторизации подставляется автоматически)
- Возвращает массив объектов `Chat`

**Файлы:**
- `src/api/chatsApi.ts` (новый)

---

## Шаг 3. Обновить chatStore

**Цель:** добавить загрузку чатов с сервера и состояния загрузки.

- Добавить в `chatStore` состояния: `loading`, `error`
- Добавить действие `loadChats()`, которое вызывает `fetchChats()` и сохраняет результат в `chats`
- Обработать ошибки: при сбое запроса сохранить сообщение в `error`

**Файлы:**
- `src/store/chatStore.ts` (изменение)

---

## Шаг 4. Обновить ChatList

**Цель:** адаптировать компонент к новой структуре данных и отображать состояния загрузки/ошибки.

- Обновить рендеринг элементов: отображать `chaName` как название чата
- Использовать `chaId` как ключ и идентификатор активного чата
- Добавить отображение индикатора загрузки
- Добавить отображение ошибки

**Файлы:**
- `src/components/ChatList.tsx` (изменение)

---

## Шаг 5. Инициировать загрузку в ChatsPage

**Цель:** запускать загрузку чатов при монтировании страницы.

- В `ChatsPage` вызвать `loadChats()` из `chatStore` при монтировании (`useEffect`)

**Файлы:**
- `src/pages/ChatsPage.tsx` (изменение)

---

## Порядок выполнения

```
Шаг 1 → Шаг 2 → Шаг 3 → Шаг 4 → Шаг 5
```
