# Authentication

## Описание

Добавление функционала аутентификации в приложение коммерческого представителя. Включает:
- единый HTTP-клиент для взаимодействия с API, с конфигурируемым `BASE_URL`
- страницу логина с аутентификацией по логину и паролю
- хранение и использование JWT-токена для авторизации запросов

## Мотивация

На данный момент приложение не имеет механизма аутентификации. Для работы с защищёнными эндпоинтами API необходимо реализовать вход пользователя и управление сессией через JWT-токен. Единый HTTP-клиент обеспечит консистентность взаимодействия с API и упростит дальнейшее развитие.

## Требования

### Конфигурация API

- Создать конфигурационный файл (`.env`) с переменной `BASE_URL`, определяющей базовый адрес API
- Реализовать единый HTTP-клиент (обёртку), который использует `BASE_URL` из конфигурации
- Все запросы к API должны проходить через этот клиент
- Клиент должен автоматически подставлять `Authorization: Bearer <token>` в заголовки авторизованных запросов

### Страница логина

- Форма с двумя полями: логин и пароль
- Кнопка отправки формы
- Валидация: оба поля обязательны для заполнения
- Отображение ошибки при неуспешной аутентификации

### Эндпоинт аутентификации

- Метод: `POST`
- URL: `{{BASE_URL}}auth/login`
- Тело запроса:
  ```json
  {
    "login": "string",
    "password": "string"
  }
  ```
- Ответ: объект, содержащий `access_token` (JWT)

### Управление токеном

- После успешного логина сохранить `access_token` в хранилище (localStorage или аналог)
- Перенаправить пользователя на основную страницу приложения
- При отсутствии токена — перенаправлять на страницу логина

### Защита маршрутов

- Неаутентифицированные пользователи не должны иметь доступ к основным страницам приложения
- При истечении или отсутствии токена — редирект на страницу логина

## Затрагиваемые области

- Конфигурация проекта (`.env`, Vite env)
- HTTP-клиент (новый модуль)
- Маршрутизация (добавление защищённых маршрутов)
- Zustand store (состояние аутентификации)
- Новая страница: Login

## Зависимости

- Axios или встроенный fetch для HTTP-клиента
- React Router (если ещё не подключён) для маршрутизации

## Вне скоупа

- Регистрация новых пользователей
- Восстановление пароля
- Refresh-токен
- Выход из системы (logout) — может быть добавлен отдельно
