# Chat List

## Описание

Получение и отображение списка доступных чатов для авторизованного пользователя. Список загружается с сервера при открытии основной страницы и отображается в компоненте `ChatList`.

## Мотивация

Для работы с чатами коммерческому представителю необходимо видеть список доступных диалогов. На данный момент компонент `ChatList` существует, но не получает данные с сервера. Нужно связать его с API и отобразить актуальный список чатов.

## Требования

### API

- Метод: `GET`
- URL: `{{BASE_URL}}chats`
- Авторизация: заголовок `Authorization: Bearer <token>` (подставляется автоматически через `httpClient`)
- Входящих параметров нет
- Ответ: массив объектов чатов в формате 
```
[
    {
        "chaCode": "1",
        "chaId": "9a11aec1-271b-417e-9f00-b625bd65afde",
        "chaName": "Серик (ТТ \"Сауле\")"
    }
]
```

### Отображение

- Список чатов отображается в компоненте `ChatList`
- Каждый элемент списка содержит название чата
- При загрузке данных отображается индикация загрузки
- При ошибке запроса отображается сообщение об ошибке
- При пустом списке отображается соответствующее сообщение

### Загрузка данных

- Список чатов загружается автоматически при монтировании страницы чатов
- Полученные данные сохраняются в Zustand store (`chatStore`)

## Затрагиваемые области

- API-слой (новая функция получения чатов)
- Компонент `ChatList` (интеграция с API)
- `chatStore` (загрузка данных с сервера)
- Страница `ChatsPage` (инициация загрузки)

## Зависимости

- `httpClient` — для выполнения авторизованных запросов к API
- `chatStore` — для хранения списка чатов
- Существующий тип `Chat` из `src/types/index.ts`

## Вне скоупа

- Пагинация списка чатов
- Поиск и фильтрация чатов
- Обновление списка в реальном времени (WebSocket)
- Отображение последнего сообщения в элементе списка
