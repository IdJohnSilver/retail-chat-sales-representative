# Send Message

## Описание

Отправка текстового сообщения в активный чат. Пользователь вводит текст в поле ввода компонента `ChatWindow` и отправляет его нажатием кнопки или клавиши Enter. Сообщение передаётся на сервер через API и добавляется в список отображаемых сообщений.

## Мотивация

Коммерческий представитель должен иметь возможность отправлять сообщения продавцам в рамках назначенных чатов. На данный момент компонент `ChatWindow` содержит поле ввода и кнопку отправки, но функция `handleSend` не реализована — сообщения не отправляются на сервер.

## Требования

### API

- Метод: `POST`
- URL: `{{BASE_URL}}chats/{{chat_code}}/messages`
- Авторизация: заголовок `Authorization: Bearer <token>` (подставляется автоматически через `httpClient`)
- Payload: `{"text": "текст сообщения"}`
- Ответ: объект отправленного сообщения

### Отправка

- При нажатии кнопки «Отправить» или клавиши Enter содержимое поля ввода отправляется через API
- После успешной отправки поле ввода очищается
- Отправленное сообщение добавляется в список отображаемых сообщений
- Пустое сообщение (только пробелы) не отправляется

### Состояния

- Во время отправки кнопка и поле ввода должны быть заблокированы для предотвращения повторной отправки
- При ошибке отправки пользователь должен увидеть уведомление об ошибке, текст сообщения не должен теряться

## Затрагиваемые области

- API-слой (новая функция отправки сообщения)
- Компонент `ChatWindow` (реализация `handleSend`)
- `chatStore` (действие отправки сообщения, добавление в список)

## Зависимости

- `httpClient` — для выполнения авторизованных запросов к API
- `chatStore` — для хранения сообщений и получения активного чата
- Тип `Message` из `src/types/index.ts`

## Вне скоупа

- Отправка вложений (файлы, изображения)
- Индикация доставки / прочтения
- Обновление сообщений в реальном времени (WebSocket)
- Повторная отправка при сбое (retry)
