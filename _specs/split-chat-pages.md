# Split Chat Pages

## Описание

Разделение функционала просмотра списка чатов и просмотра сообщений чата на две отдельные страницы с собственными маршрутами. Список чатов становится отдельной страницей `/chats`, а просмотр сообщений конкретного чата — отдельной страницей `/chat/{code}`, где `{code}` — уникальный код чата (`chaCode`). На обеих страницах в шапке размещается кнопка «Назад», использующая браузерную историю для возврата на предыдущий шаг.

## Мотивация

Сейчас список чатов и окно сообщений отображаются одновременно на одной странице в режиме «мастер-детейл». Такая компоновка неудобна на мобильных устройствах и не даёт возможности:

- делиться прямой ссылкой на конкретный чат
- использовать кнопку «Назад» браузера для навигации между списком и чатом
- открывать чат в новой вкладке

Разделение на отдельные страницы с url-навигацией делает приложение более согласованным с привычными паттернами веб-навигации и упрощает последующую адаптацию под мобильные клиенты.

## Требования

### Маршрутизация

- Страница списка чатов доступна по маршруту `/chats`
- Страница просмотра сообщений доступна по маршруту `/chat/:code`, где `:code` — `chaCode` активного чата
- Обе страницы защищены авторизацией (так же, как текущие защищённые маршруты)
- При переходе на несуществующий или недоступный `code` пользователь должен получить понятное состояние (сообщение об ошибке или редирект на список чатов)
- Маршрут по умолчанию после входа в систему ведёт на `/chats`

### Страница списка чатов `/chats`

- Отображает список чатов (как в текущем компоненте `ChatList`)
- Клик по чату выполняет навигацию на `/chat/{chaCode}` вместо установки активного чата в рамках одной страницы
- В шапке отображается кнопка «Назад», выполняющая переход на один шаг назад по браузерной истории

### Страница просмотра сообщений `/chat/:code`

- Отображает окно сообщений (как в текущем компоненте `ChatWindow`) для чата с `chaCode`, равным параметру маршрута
- При загрузке страницы сообщения чата запрашиваются с сервера на основе `code` из URL
- Активный чат в `chatStore` синхронизируется с параметром маршрута
- В шапке отображается кнопка «Назад», возвращающая пользователя по браузерной истории (как правило, на `/chats`)

### Кнопка «Назад»

- Размещается в шапке на страницах `/chats` и `/chat/:code`
- Использует браузерную историю (шаг назад), а не жёстко заданный маршрут
- Если истории нет (прямой вход по ссылке), кнопка должна вести на разумный маршрут по умолчанию (`/chats`)

## Затрагиваемые области

- Роутинг приложения (конфигурация защищённых маршрутов)
- Компонент `ChatList` (переход на страницу чата вместо локального выбора)
- Компонент `ChatWindow` (получение `chaCode` из параметра маршрута, загрузка сообщений по URL)
- `chatStore` (синхронизация активного чата с маршрутом)
- Компонент шапки/layout страниц (кнопка «Назад»)

## Зависимости

- Библиотека маршрутизации (например, `react-router-dom`), уже используемая в проекте для защищённых маршрутов
- `chatStore` — для хранения состояния чатов и сообщений
- `httpClient` — для запроса сообщений по `chaCode` из URL

## Вне скоупа

- Редизайн интерфейса списка чатов или окна сообщений
- Адаптивная вёрстка под мобильные устройства
- Навигация с помощью жестов (свайпов)
- Глубокие ссылки на конкретное сообщение внутри чата
- Сохранение прокрутки и состояния ввода при возврате на страницу чата
